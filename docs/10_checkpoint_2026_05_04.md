## Bug Encontrado

Pergunta:

```text
qual foi a receita deste mes
```

Erro no BigQuery:

```text
No matching signature for operator = for argument types: DATE, STRING
```

Causa:

```text
O agente gerou uma comparacao entre campo DATE e string.
Exemplo ruim: mes = '2026-05-01'
Exemplo correto: mes = DATE '2026-05-01'
```

## Correcao Aplicada No Codigo

Arquivos alterados:

- `finance_ai_agent/agent.py`
- `finance_ai_agent/tools/bigquery_tool.py`
- `chat_adapter/main.py`

Mudancas:

- O prompt agora orienta o agente a usar `DATE 'YYYY-MM-DD'` para campos de data.
- Para "este mes", "mes atual", "agora" e termos equivalentes, o agente deve buscar `MAX(mes)` antes de responder.
- Se o mes calendario atual nao existir nos dados ficticios, o agente deve usar o ultimo mes disponivel e informar isso.
- A tool de BigQuery agora captura erros `BadRequest` e retorna `status: erro` para o agente em vez de derrubar o Cloud Run.
- O adapter agora divide mensagens com multiplas perguntas e chama o agente uma vez para cada pergunta, consolidando as respostas no Google Chat.
